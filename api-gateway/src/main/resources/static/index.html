System Map

How the platform works end to end.

This application runs as a Spring microservice platform. Requests enter through the gateway, configuration is supplied centrally, services register with Eureka, and orders complete through RabbitMQ-driven inventory processing.

Entry Layer

Client and Platform Entry

User UI / Postman Clients access the system through the gateway.
→
API Gateway Routes `/auth`, `/products`, `/orders`, `/inventory`.

Control Layer

Platform Services

Config Server Supplies shared and service-specific configuration.
Service Registry Eureka discovery for runtime service lookup.
RabbitMQ Async order and inventory messaging backbone.

Business Services

Domain Flow

Auth Service Register, login, RSA JWT issuance, role handling.
Product Service Catalog management and inventory-backed product views.
Order Service Creates orders, calculates totals, publishes stock requests.
Inventory Service Owns stock and processes reservation / reduction logic.

Data Stores

Service-Owned Databases

auth_db Users and roles.
product_db Catalog data.
inventory_db Stock and processed message tracking.
order_db Orders and outbox events.

Order Flow

Request Lifecycle

1
User authenticates

Auth Service validates credentials and returns an RSA-signed JWT.

2
Gateway routes the request

The gateway forwards product, inventory, and order requests to discovered services.

3
Order Service creates a pending order

It calculates total price from product data and records an outbox event.

4
RabbitMQ carries the stock request

Inventory Service consumes the message and applies stock rules safely.

5
Inventory result updates the order

The order becomes `CREATED` on success or `CANCELLED` if stock is unavailable.
